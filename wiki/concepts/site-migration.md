# Site Migration (SEO)

**Definition:** Making major changes to a site's tech, structure, design or location. Usually hurts SEO short-term and helps long-term; a checklist (esp. redirect mapping) minimises the damage. → [[semrush-site-migration]]

## What we know
- **Common reasons:** new framework/platform, better architecture, **HTTP→HTTPS** (Google-recommended), new server (speed), domain-name change (rebrand). → [[semrush-site-migration]]
- **SEO impact:** negative short-term (engines need to re-understand), positive long-term if genuinely better. → [[semrush-site-migration]]
- Minimise damage with migration best practices — chiefly mapping old→new URLs with redirects.

## Relations
- Relies on [[301-redirect]] (permanent) + [[302-redirect]] (temporary); domain change overlaps [[exact-match-domain]].
- Part of technical SEO ([[seo-audit]], [[http-status-codes-seo]]).
- UX/brand sibling: [[website-redesign]] (same 301-redirect risk on content migration; redesign adds UX/IA/brand layers).
