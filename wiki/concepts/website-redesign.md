# Website Redesign

**Definition:** A strategic rethink and reorganization of a brand's digital experience — not a cosmetic lifting. Done without strategy it loses organic traffic, rankings, and conversions → [[website-redesign]].

## Two forms
- **Visual refresh** — solid base, dated look (colors/fonts/visuals/nav).
- **Complete redesign** — structural problems → new information architecture, content, technical integrations.
→ [[website-redesign]]

## Strategy first
- Align objectives (leads / mobile / identity / accessibility), understand users, assess SEO impact before touching a pixel → [[website-redesign]].

## 8-step process
1. Full audit ([[seo-audit]], behavior/bounce/exits) → 2. Measurable goals (+20% leads, <2 s) → 3. User profile/needs (heatmaps, usability tests) → 4. Clear IA (menu hierarchy, friendly URLs) → 5. Content strategy review (brand voice, SEO) → 6. UX + accessibility ([[core-web-vitals]], [[mobile-seo]]) → 7. Test in real conditions ([[ab-testing]]) → 8. Measure over time ([[google-analytics-4]], [[google-search-console]]) → [[website-redesign]].

## Common mistakes
- Aesthetics-only; ignoring existing data; content migration **without 301 redirects** ([[301-redirect]]) → SEO loss; poor cross-team communication → [[website-redesign]].

## Relations
- UX/brand sibling of [[site-migration]] (shares URL-redirect risk); audited via [[seo-audit]]; speed/UX = [[core-web-vitals]] + [[mobile-seo]].
- Brand consistency = [[brand-identity]]; conversion focus ties to [[landing-page]].
