# YouTube Algorithm

**Definition:** YouTube's [[machine-learning]] system that decides which videos appear for each user across Search, the recommendations feed, the Trending tab, and Shorts. It acts as a personalized curator whose objective is to **maximize total time spent on the platform** → [[youtube-algorithme]].

## What we know
- **Ranking signals** → [[youtube-algorithme]]: watch time, [[click-through-rate]] (thumbnails/titles), engagement (comments/likes/shares/subs), audience retention, and user history/personalization.
- **Surface-specific logic** → [[youtube-algorithme]]:
  - *Search* = relevance (keywords in title/desc/tags) + channel authority + user history + popularity + recency.
  - *Recommendations* = predict the next video via similarity, similar-user behavior, engagement velocity, trends.
  - *Trending* = speed/consistency of engagement (not absolute views); small channels can appear; local relevance; format diversity.
  - *Shorts (2025)* = rapid repeated interactions, retention, rewatches, immediate engagement; sounds + auto-captions help.
- **Creator levers** → [[youtube-algorithme]]: quality content, hook in first seconds, video series (cumulative watch time), community interaction, captions, metric tracking (CTR/retention), fast early engagement for Trending.

## Relations
- Subtype of [[machine-learning]]-driven recommendation; sibling logic to [[ai-marketing]] recommendation engines.
- The ranking/distribution engine for [[video-marketing]] (YouTube hosting = SEO) and part of [[social-media-strategy]] / [[search-everywhere-optimization]].
- Thumbnails/titles connect to [[click-through-rate]], [[headline]], [[hook]].
