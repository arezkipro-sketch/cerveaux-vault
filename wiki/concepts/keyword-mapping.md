# Keyword Mapping (Carte de Mots Clés)

**Definition:** The process of assigning keyword groups to specific pages on a site — 1 URL = 1 primary keyword intent. Keyword mapping creates an explicit link between a site's information architecture and its target search queries.

## What we know

- **Process:**
  1. Research all target keywords (head + long tail)
  2. Group keywords by shared intent/topic (not just similar words)
  3. Match each group to an existing or planned page
  4. Ensure no two pages target the same keyword → prevents [[keyword-cannibalization]]
- **Output:** a spreadsheet or visual mindmap of URL ↔ keyword group ↔ intent
- Keyword mapping is the bridge between [[keyword-research]] and [[topic-clusters]] / [[cocon-semantique]] — you know your keywords; now you know which page earns each one → [[semrush-seo-checklist-41]]
- Tools: Semrush Keyword Strategy Builder, Google Sheets; organize by category then by intent

## Relations
- Feeds into [[topic-clusters]] / [[cocon-semantique]] (which page is "pillar" vs "cluster").
- Prerequisite for [[maillage-interne]]: you can't anchor correctly if you don't know which page owns which keyword.
- Detects [[keyword-cannibalization]] before it happens — two pages mapped to the same query = problem to fix upstream.
- Outputs inform [[article-structure]] and [[seo-roadmap]] (what to create next, what to merge/redirect).

## Tensions / open questions
- At what granularity do you split groups? One page per sub-intent, or group related sub-intents on one page?
- How do you handle keyword groups where search intent is genuinely mixed?
