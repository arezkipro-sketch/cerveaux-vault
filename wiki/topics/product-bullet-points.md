# Product Bullet Points (E-commerce Copy)

**Thesis:** A product bullet should not stop at the spec. The converting pattern is **Feature → Benefit** in one line: state the technical fact, then translate it into what the buyer gets. This satisfies SEO (enumeration lists), AI absorption (numbers/comparisons), and conversion (emotion + proof) at once.

## The rule (synthesis of sources)

1. **Lead with the feature, close with the benefit.** FAB says "people don't buy a product, they buy solutions" → [[copywriting-frameworks]]. AIDA's Desire step literally says bullets exist "to highlight the benefits" → [[copywriting-frameworks-2026]].
   - ❌ `Matière : PVC transparent`
   - ✅ `Matière PVC transparent — contenu visible en un regard, fini de fouiller le sac`

2. **Put a number in at least one bullet.** Numbers lift both human credibility ("97% reported…") and AI absorption (+61.55% influence) → [[citation-absorption]], [[copywriting-psychology]]. Dimensions, capacity, count of compartments, inches.

3. **Keep the 4 C's: Clear, Concise, Compelling, Credible.** No jargon, one idea per bullet → [[copywriting-frameworks]].

4. **Use bullets for enumeration only** (specs, what's-in-the-box). Prose/headings carry the story → [[article-structure]].

5. **Ethics gate:** no exaggeration, claim only what's true → [[copywriting-psychology]], [[e-e-a-t]].

## Quick scorecard for auditing a bullet block
| Check | Pass condition |
|---|---|
| Feature→Benefit | ≥3 of N bullets translate spec into outcome |
| Number present | ≥1 bullet has a concrete figure |
| Concise | each bullet ≤ ~12 words |
| No vague filler | no "Atout : pratique / qualité" without payload |
| Honest | no unverifiable superlative |

## Tensions / open questions
- Optimal bullet count per product page — no source gives a number; working default 4–6.
- FAQ-style blocks: weak for AI absorption [[citation-absorption]] but still useful for SEO snippets — keep below the fold.
