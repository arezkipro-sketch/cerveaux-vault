# HTTP Errors and SEO Impact — X Thread (JotaroSeo)

**One-liner:** An 8-part French X thread explaining how HTTP status codes (4xx, 5xx) hurt crawling, indexation, user experience, and link juice — and how to monitor and fix them.

## Key takeaways
- HTTP error codes (especially 404 and 500) are **negative signals** to search engines — they indicate site problems → [[http-status-codes-seo]].
- Broken pages create **crawl budget waste**: robots skip inaccessible pages, leaving them unindexed → [[http-status-codes-seo]].
- 404s on inbound-linked pages **kill link juice** — the authority transfer to the target page is lost → [[link-juice]].
- UX damage from errors → higher bounce rate → lower perceived quality by search engines. **SXO** (SEO + UX) is the proposed remedy.
- HTTP code families: 1xx (info), 2xx (success), 3xx (redirects), 4xx (client error), 5xx (server error).
- **Practical fix**: monitor with Screaming Frog or Google Search Console (GSC). 301 redirect dead 404s to relevant pages — but **not all 404s need redirecting** (some pages are legitimately gone).
- Resolving HTTP errors = technical SEO practice signalling site accessibility to engines.

## Notable claims (with locations)
