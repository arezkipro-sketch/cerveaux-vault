# Comment écrire un article de blog optimisé SEO en 2026

Guide pratique de Sédestral (outil SaaS content/SEO FR) couvrant les 5 étapes de la création d'un article de blog optimisé : mots-clés, structure HTML, contenu/E-E-A-T, balises meta/technique, maillage interne et suivi post-publication.

## Overview

Guide orienté praticien, bien plus actionnable que la plupart des articles de référence sur le sujet. Particulièrement utile pour : le tableau CTR des balises, la méthode PAS en introduction, la règle de densité 1-2 %, et la logique "positions 11-20 = gains rapides".

Source commerciale (Sédestral vend un outil SEO) mais le contenu reste générique et applicable sans l'outil.

## Key Takeaways

1. **Cibler les positions 11-20** pour des gains rapides sans affronter les gros sites — quick wins avant d'attaquer les tops.
2. **Minimum 300 recherches/mois** pour un mot-clé principal — en dessous, l'audience potentielle ne justifie pas l'effort.
3. **Champ sémantique = 3 catégories** : principal + secondaires + questions. Chaque sous-titre peut contenir un terme du champ.
4. **H1 unique, 50-60 chars, mot-clé principal dedans** ; H2 = sections majeures ; H3/H4 = détails. Pas de sauts de niveau.
5. **PAS en introduction** (Problème → Agitation → Solution) + mot-clé principal dans les 100 premiers mots.
6. **Densité mot-clé 1-2 %** = environ 15-30 occurrences pour 1 500 mots.
7. **Title tag optimisé = CTR +30-50 %** ; meta description = +15-25 % (données Sédestral).
8. **Images** : WebP/AVIF, <150 Ko, alt descriptif avec mot-clé, lazy-loading → améliore LCP.
9. **Mise à jour trimestrielle** — les articles vieillissent et perdent du trafic face aux contenus plus récents.
10. **Tension avec JotaroSeo** : Sédestral recommande 2-3 liens internes minimum par article ; JotaroSeo recommande 1-2 maximum → voir [[maillage-interne]].

## Detailed Notes

### Section 1 — Mots-clés

**Critères de sélection du mot-clé principal :**
- Volume ≥ 300 recherches/mois (intérêt authentique garanti)
- Difficulté modérée
- Intention claire : informationnel / navigationnel / transactionnel

**Stratégie quick wins :**
- Cibler les mots-clés en positions 11-20 → page 2 → page 1 avec moins d'effort que partir de zéro
- Éviter d'affronter les sites très puissants dès le début

**Construction du champ sémantique :**

| Catégorie | Source | Utilisation |
|---|---|---|
| Mots-clés secondaires | Semrush / Ahrefs | H2, sous-titres, début de paragraphes |
| Synonymes / variantes | 1.fr, YourTextGuru | Corps du texte, richesse sémantique |
| Questions | AnswerThePublic, PAA Google | Sous-titres H3, section FAQ |

- Densité cible : 1-2 % (= 15-30 occurrences pour 1 500 mots)
- Intégrer naturellement : si une phrase sonne artificielle → synonyme

### Section 2 — Structure HTML

**Règles de balisage :**

| Balise | Règle | Contenu |
|---|---|---|
| H1 | 1 unique par page, 50-60 chars | Mot-clé principal, accrocheur |
| H2 | Sections principales | Terme du champ sémantique |
| H3/H4 | Sous-parties | Variantes longue traîne |
| Strong | Tous les 50-100 mots | Points essentiels uniquement |

- **Pas de sauts de niveau** (H2 → H4 directement = interdit)
- Paragraphes : 2-4 phrases, 1 idée par paragraphe
- Listes à puces → augmentent les chances de **featured snippet**
- Tableaux → appréciés pour les **extraits enrichis**
- Image / code / citation tous les 3-4 paragraphes → engagement

**Table des matières** (pour articles > 1 500 mots) :
- Avec ancres HTML = liens internes intra-page
- Aide Google à analyser la structure + navigation rapide pour le lecteur

**Schema.org :**
- `Article` pour articles standards
- `FAQPage` pour les sections Q&R
- Tester avec l'outil de validation Google

### Section 3 — Contenu et E-E-A-T

**Introduction (100 premiers mots) :**
- Méthode **PAS** : Problème → Agitation → Solution
- Mot-clé principal en apparition naturelle dès le début

**Longueur cible :**
- 1 500-2 000 mots minimum
- Trop court = sujet effleuré ; trop long = ennuyeux. Trouver l'équilibre.

**E-E-A-T en pratique :**

| Signal | Exemples concrets |
|---|---|
| Données chiffrées | "63 % des internautes" (pas "beaucoup") |
| Études de cas | Résultats mesurables + chiffres |
| Citations d'experts | Professionnels reconnus du domaine |
| Liens vers autorités | Google, Moz, HubSpot, Semrush |

**CTA en conclusion** aligné sur l'objectif commercial.

### Section 4 — Balises meta et technique

**Impact CTR des balises (données Sédestral) :**

| Élément | Longueur idéale | Contient le mot-clé | Impact CTR |
|---|---|---|---|
| Title tag | 50-60 chars | Oui, dès le début | +30 à +50 % |
| Meta description | 155-160 chars | Oui, naturellement | +15 à +25 % |
| URL | Court, clair | Oui si possible | +5 % |
| H1 | Unique, 50-60 chars | Oui, c'est l'essence | Classement Google |

- Si CTR < 2 % une semaine après publication → retravailler title/meta
- Google réécrit la meta dans >70 % des cas → rédiger quand même pour influencer

**Images :**
- Format : WebP ou AVIF
- Poids : < 150 Ko
- ALT : descriptif + mot-clé pertinent
- Nom de fichier : `optimisation-article-seo.webp` (pas `IMG_2024.jpg`)
- Lazy-loading : gain 1-2 secondes sur LCP
- TinyPNG / Squoosh.app : réduction 50-70 % sans perte visible
- CDN + cache navigateur pour visiteurs récurrents

**Core Web Vitals (score cible : vert > 90 sur mobile ET desktop) :**
- LCP < 2,5 s
- CLS : stabilité visuelle
- INP : réactivité aux interactions

**Vérifications techniques complémentaires :**
- Canonical tag → prévenir contenu dupliqué
- robots.txt + sitemap.xml → aucune page importante bloquée

### Section 5 — Maillage interne et suivi post-publication

**Maillage interne :**
- 2-3 liens internes minimum par article *(⚠ tension avec JotaroSeo qui recommande 1-2 max)*
- Ancres descriptives avec mot-clé secondaire
- Relier articles → pages piliers / fiches produit → cohérence sémantique
- 1 lien externe vers source fiable → renforce crédibilité E-E-A-T
- Audit régulier (Screaming Frog) → liens cassés + pages orphelines

**Suivi post-publication (GSC, hebdomadaire) :**
- Positions, impressions, CTR
- Position 11-20 → optimiser → passer en page 1 (quick win)
- CTR < 2 % → retravailler title/meta
- Mots-clés qui stagnent → enrichir (+ quelques centaines de mots, reformuler sections)

**Fraîcheur du contenu :**
- Mettre à jour tous les trimestres
- Intégrer données récentes
- Google favorise la fraîcheur → contenu non mis à jour perd du trafic progressivement
