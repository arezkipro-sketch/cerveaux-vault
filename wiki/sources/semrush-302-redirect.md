# Redirection 302 — Summary

**One-liner:** A 302 is a *temporary* redirect — like an out-of-office message — sending users/bots to another page while preserving the original page's keyword rankings and link value.

## Key takeaways
- **302 (Found) = temporary**; **301 = permanent**. Use 302 when the move is temporary (A/B test, maintenance, page under construction, seasonal/regional swap) so the original keeps its ranking.
- Improves UX by not landing users on broken/under-construction pages.
- **Mistake:** using 302 for a *permanent* move — link equity then doesn't fully consolidate to the new URL; use 301 instead.
