# Migration de site web — Summary

**One-liner:** Major site changes (platform/structure/HTTP→HTTPS/server/domain) usually hurt SEO short-term, help long-term — a checklist minimises the damage.

## Key takeaways
- **What:** major changes to a site's tech, structure, design or location, to improve UX + SEO. Hard process — need a good reason.
- **Common reasons:** new framework/platform (WordPress/Wix), better architecture, HTTP→HTTPS (Google-recommended security), new server (speed), domain name change (rebrand).
- **SEO impact:** negative short-term (engines need time to re-understand), positive long-term if the change is genuinely better. Minimise damage via best practices (notably redirect mapping).
