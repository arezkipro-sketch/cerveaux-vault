# Website redesign — Summary

**One-liner:** A website redesign is a strategic rethink of the brand's digital experience (not a cosmetic lifting) — done without strategy it loses organic traffic, rankings, and conversions; this guide covers the two redesign types, common mistakes, and an 8-step process.

## Key takeaways
- **Two forms:** **visual refresh** (solid base, dated look — colors/fonts/visuals/nav) vs **complete redesign** (structural problems → new information architecture, content, technical).
- **Strategy before pixels:** align objectives (leads? mobile? identity? accessibility?), understand users, assess SEO impact. "Redesign without strategy = darts blindfolded."
- **Key considerations:** UX, mobile compatibility, **load speed** (→ SEO + satisfaction), prior audit, brand consistency, security, ease of updating.
- **Common mistakes:** aesthetics-only (ignoring performance/journey); redesign without analyzing existing data → SEO/conversion loss; content migration **without proper 301 redirects** → SEO harm; poor cross-team communication.

## 8-step process (with locations)
1. **Full audit** of existing site (bounce, exit points, top organic content) → 2. **Precise measurable goals** (+20% leads, <2 s load) → 3. **User profile/needs** (heatmaps, usability tests, interviews) → 4. **Clear IA structure** (menu hierarchy, fewer clicks, friendly URLs) → 5. **Review content strategy** (brand voice, SEO, new formats) → 6. **UX + accessibility** (mobile buttons, color contrast, speed) → 7. **Test in real conditions** (browsers/screens/speeds; A/B) → 8. **Measure over time** (GA, GSC, heatmaps) *(§étapes)*.
